# CLI Help Enrichment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Architecture:** Keep the command surface unchanged and enrich only the Clap metadata in `crates/numi-cli/src/cli.rs`. Drive the change through TDD by extending the existing CLI help integration tests first, then implementing the minimum metadata needed to satisfy those assertions.

**Tech Stack:** Rust, Clap 4, existing CLI integration tests

    name = "numi",
    version,
    about = "Generate Swift code from Apple project resources",

    after_help = "Examples:\n  numi init\n  numi generate\n  numi check\n  numi generate --workspace\n  numi dump-context --job l10n",
    propagate_version = true,
    subcommand_required = true
)]

- [ ] **Step 2: Add command-level descriptions and examples**

Add Clap `#[command(...)]` metadata on the subcommands in `Command` and `ConfigSubcommand` so they read like:

```rust
#[derive(Debug, Subcommand)]
pub enum Command {
    #[command(about = "Generate outputs for one config or workspace")]
    Generate(GenerateArgs),
    #[command(about = "Check whether generated outputs are up to date")]
    Check(CheckArgs),
    #[command(about = "Write a starter numi.toml in the current directory")]
    Init(InitArgs),
    #[command(about = "Inspect resolved config paths and values")]
    Config(ConfigCommand),
    #[command(
        name = "dump-context",
        about = "Print the template context for a single job",
        after_help = "Examples:\n  numi dump-context --job l10n\n  numi dump-context --config AppUI/numi.toml --job assets"
    )]

}
```

- [ ] **Step 3: Add flag help text and per-command examples**

Add `help = "...”` metadata for the user-facing flags, and add `after_help` example blocks on `GenerateArgs`, `CheckArgs`, and `DumpContextArgs` using wording that matches the approved spec. Use these concrete strings:

```rust
#[derive(Debug, Args)]
#[command(
    after_help = "Examples:\n  numi generate\n  numi generate --job assets --job l10n\n  numi generate --workspace"
)]
pub struct GenerateArgs {
    #[arg(long = "config", help = "Use a specific numi.toml instead of auto-discovery")]
    pub config: Option<PathBuf>,
    #[arg(
        long = "workspace",
        action = ArgAction::SetTrue,
        help = "Use the ancestor workspace manifest instead of the nearest member manifest"
    )]
    pub workspace: bool,
    #[arg(long = "job", help = "Limit generation to the selected job name")]
    pub jobs: Vec<String>,
    #[command(flatten)]
    pub incremental_override: IncrementalOverrideArgs,
}

#[derive(Debug, Args)]
#[command(after_help = "Examples:\n  numi check\n  numi check --job l10n\n  numi check --workspace")]
pub struct CheckArgs {
    #[arg(long = "config", help = "Use a specific numi.toml instead of auto-discovery")]
    pub config: Option<PathBuf>,
    #[arg(
        long = "workspace",
        action = ArgAction::SetTrue,
        help = "Use the ancestor workspace manifest instead of the nearest member manifest"
    )]
    pub workspace: bool,
    #[arg(long = "job", help = "Limit checking to the selected job name")]
    pub jobs: Vec<String>,
}

#[derive(Debug, Args)]
pub struct InitArgs {
    #[arg(long, help = "Overwrite an existing numi.toml in the current directory")]
    pub force: bool,
}

#[derive(Debug, Args)]
#[command(
    after_help = "Examples:\n  numi dump-context --job l10n\n  numi dump-context --config AppUI/numi.toml --job assets"
)]
pub struct DumpContextArgs {
    #[arg(long = "config", help = "Use a specific numi.toml instead of auto-discovery")]
    pub config: Option<PathBuf>,
    #[arg(long = "job", help = "Job name to render as JSON context")]
    pub job: String,
}

#[derive(Debug, Args, Default, Clone, PartialEq, Eq)]
pub struct IncrementalOverrideArgs {
    #[arg(
        long = "incremental",
        action = ArgAction::SetTrue,
        conflicts_with = "no_incremental",
        help = "Force incremental parsing when supported"
    )]
    pub incremental: bool,
    #[arg(
        long = "no-incremental",
        action = ArgAction::SetTrue,
        help = "Disable incremental parsing even when the config enables it"
    )]
    pub no_incremental: bool,
}
```
